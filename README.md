I am working on a video and audio quality framework, it utilizes ffmpeg for video transmission. Has an optional plugin system, that lets a user choose which encryption and key distribution scheme to use, the plugins should be implemented by users so a scheme would be nice. Furthermore, it gets the initial video and compares it to the transmitted one, thus utlizing the full-refence approach. The system should be able to be deployed both remotely on two devices, thus config.yaml will point to what device it is on. No metrics regarding the internet connection so jitter, etc should be derived. The framework should solely focus on the loss caused by the encryption scheme and delays caused by it.
